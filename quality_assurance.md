# Quality Assurance

## Editorial Review Process

Before delivering any report, perform a comprehensive editorial review with the perspective of a 20+ year expert editor and publicist:

- **Structure**: Verify internal references exist, cross-references accurate, section names match
- **Efficiency**: Eliminate redundancy, remove unnecessary preambles/postambles, consolidate related info
- **Audience**: Confirm technical depth matches audience, terminology appropriate, examples suit sophistication
- **Polish**: Fix typos/grammar/phrasing, ensure consistent tone, verify clean formatting, check logical progression
- **Evidence**: Validate file paths/line numbers/code refs, SQL queries/DB fields, ticket links, data consistency; for complex work cross-check against notes
- **Clarity**: Read as if first time, identify ambiguities, flag unclear sections, ensure unambiguous action items

**Key Question:** "Would a busy professional in the target audience understand this completely on first read, without confusion or missing context?"

## Pre-Delivery Checklist

Use this checklist before submitting any report:

- [ ] **Audience Alignment**: Technical depth matches target audience
- [ ] **Completeness**: All required sections present
- [ ] **Ticket Reference**: Ticket number/link included (if applicable)
- [ ] **Permissions Documented**: Required configurations, settings, permissions noted
- [ ] **Visual Elements**: Diagrams add clarity and follow style guidelines
- [ ] **Heading Format**: All major headings in Title Case, grammar correct
- [ ] **Length Target**: Within expected range for report type
- [ ] **Typos & Grammar**: Proofread for errors
- [ ] **Action Items**: Next steps clearly identified (if applicable)
- [ ] **MCP Context**: Atlassian/Jam tools used appropriately for context gathering
- [ ] **Notes Review**: For complex work, review thinking notes for completeness

**Note:** Common mistakes for all report types are documented in agentic://references/common_mistakes. Success criteria are in agentic://references/success_criteria.
