# Report Iteration & Feedback

When revising a report based on feedback:

1. **Identify gap**: What information or clarity is missing?
2. **Assess audience fit**: Is the technical depth still appropriate?
3. **Add, don't replace**: Expand sections rather than rewriting unless explicitly requested
4. **Version naming**: Use `_v2`, `_v3` suffixes for significant revisions
5. **Document changes**: Add brief "Revision History" section at bottom if helpful
